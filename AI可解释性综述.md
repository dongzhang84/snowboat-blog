# 可解释性综合 Review · 提纲

> 这是一份**提纲**，不是成文。每节下面标注的是"这一节要论证/讲清什么"，用于讨论结构、判断取舍。

---

# 贯穿全文的三个设计决定

- **(a) 沿问题走，不沿方法走**——方法会过时，几个核心问题不会。
- **(b) 把"忠实性"提到很靠前**——它是判断后面一切方法的标尺，不能塞到最后当注脚。
- **(c) 经典派与机制派并重**——避免写成"只有 LLM mech interp"的偏科综述。

# 贯穿全文的一条主线（本轮新增，最重要）

整篇 Review 真正要回答的总问题不是"有哪些方法"，而是：
**"理解了模型，然后呢？这种理解是真的，且有用吗？尤其能不能反哺模型本身？"**
后面每一节都要能回扣到这条主线——具体落点就是第 2 节定义的"用处阶梯"：评价任何一项工作，都问"它把我们在这个阶梯上往上推了吗"。

---

# 0. 摘要 + 范围声明
说清覆盖什么、不覆盖什么（聚焦"理解模型内部机制与决策依据"，不展开公平性/隐私等相邻话题），并给出全文组织逻辑。

# 1. 引言：为什么需要可解释性
- 1.1 三种动机（工程调试 / 信任与合规 / AI 安全对齐），并说明动机如何决定方法选择——这是全领域分裂的根源。
- 1.2 核心张力预告：能力进步 vs. 理解滞后（Amodei 论点），引出紧迫性。
- 1.3 本文核心主张（提前亮观点，便于读者带着判断读下去）。

# 2. 核心追问：理解了，然后呢？（本轮新增 · 全文主线）
> 这一节把"是不是真的更懂了 + 有什么用 + 能否反哺训练"立为全文骨架。

- **2.1 可解释性真的带来理解了吗？**（分层、诚实地回答，与第 4 节"忠实性"挂钩）
  - 小模型/孤立任务：是，能完整逆向且因果验证（induction head、两位数加法电路）。
  - 前沿大模型整体：远未——多是"在巨大黑箱上开几个小窗"。
  - 反向风险：**制造虚假的"懂"**——看似合理却不忠实的解释。故"更懂了吗"必须与"忠实吗"绑定回答。
- **2.2 理解的用处阶梯（从弱到强，全文回扣的标尺）**
  1. **诊断**（最弱、最成熟）：看懂为什么错/有偏差，纯观察，工业界常用。
  2. **监控**：运行时实时盯内部（欺骗、谄媚特征是否亮），给模型装仪表盘。
  3. **编辑 / 干预**：直接改内部（model editing/ROME；金门大桥实验为极端演示），不重训改行为。
  4. **指导训练**（最接近"反哺训练"，最不成熟）：发现坏机制后改数据/目标；把"内部不该出现某特征"写进训练目标；用它审计新模型决定是否部署。
  5. **安全保证**（终极愿景，目前基本做不到）：部署前从内部"证明"模型不会做某类危险事——mech interp 的圣杯。
- **2.3 残酷现状**：阶梯越往上落地越少。第 1、2 层在用；第 3 层研究里可行、未成生产标配；第 4 层多为愿景与零星探索；主流大模型训练**目前基本没靠 mech interp 驱动**。"已在帮助训练"是夸大。
- **2.4 由此确立全文判断标准**：每项工作都问"它把我们在阶梯上往上推了吗"。原"应用"一节因此不再是孤立罗列，而是这条主线的展开。

# 3. 概念地基：什么叫"解释"，以及术语之乱
- 3.1 厘清被混用的词：interpretability / explainability / XAI / transparency。
- 3.2 两条切分轴：事后 vs. 内在；局部 vs. 全局。画一张分类地图。
- 3.3 "解释单位"这个关键概念：输入特征 → 内部表示/电路，为后文范式转移埋线。

# 4. 元问题：忠实性 vs. 可信度（全文的标尺）
- 4.1 定义两者，论证它们常常错位，且"让人安心的解释更危险"。
- 4.2 反面证据：sanity check（权重随机化后 saliency 不变）等。
- 4.3 评判框架：每出现一个方法，都用"它忠实吗，还是只让我安心"去拷问。

# 5. 经典 / 归因式可解释性（成熟、低调、工业主流）
- 5.1 内在可解释模型：线性/逻辑回归、决策树、GAM——"模型即解释"。
- 5.2 事后归因：permutation importance、LIME、SHAP（Shapley 值的博弈论基础）、saliency。
- 5.3 反黑箱立场：Rudin——高风险场景是否应拒绝事后解释、直接用透明模型。
- 5.4 局限：高维、强非线性、尤其 LLM 上为何失效——承上启下。

# 6. 范式转移：为什么 LLM 逼出了一套新东西
- 6.1 解释单位的崩塌："第 47 个 token 重要"为何等于没说。
- 6.2 有意思的计算在内部、不在输入层——从"看外面"转向"看里面"的必然性。

# 7. 机制可解释性（前沿、高调、尚不成熟）——本文重头
- 7.1 难题：多义神经元（polysemantic），为何"读神经元"行不通。
- 7.2 成因：superposition（叠加），用"灯泡组合"直觉解释。
- 7.3 线性表示假设：概念是激活空间里的方向（特征向量）。
- 7.4 拆解工具：稀疏自编码器（SAE）及其变体（transcoder 等），把叠加摊成单义特征。
- 7.5 从特征到计算：circuit、attribution graph、induction head——"模型在算什么"。
- 7.6 高层结构：规划/前瞻、多语言共享表示、persona vector（人格向量）、内省。
- 7.7 因果验证：activation patching、特征干预（金门大桥实验作为忠实性范例）。
- 7.8 **每个子方向都回扣第 2 节阶梯**：它停在诊断，还是已经能编辑/指导训练？

# 8. 中间地带
probing（探针）、attention 分析（及"权重≠重要性"的忠实性教训）、反事实方法——填补经典派与机制派之间的谱系。

# 9. 评估：怎么衡量一个解释好不好
> 很多 Review 缺这一节，但它最该有——也直接解释了第 11 节"为什么这么难"。
- 9.1 忠实性度量、稳定性、可信度的人评。
- 9.2 因果类评测、benchmark（如电路定位 shared task）。
- 9.3 自动化解释的可信度悖论：用模型解释模型，谁来解释那个解释者。

# 10. 应用与落地
诊断与数据偏差发现、模型编辑、安全监控（谄媚/幻觉/欺骗的检出）、对齐审计、合规。强调"理解在先、管控在后"的逻辑顺序，破除"可解释性=管控"的误解。（即第 2 节阶梯在现实中的落点。）

# 11. 这个领域为什么这么难：结构性困境（本轮新增）
> 不是吐槽，是要论证：这些困难多数是**结构性的、非努力可解**，从而支撑第 13 节的开放问题，并给后来者一个清醒的预期。
- 11.1 **验证地狱**：几乎无法证明一个解释为真，只能找到自洽证据、难以证伪——付出巨大却拿不到确定性。（直接源自第 4 节忠实性。）
- 11.2 **superposition 是地基性诅咒，非可攻克难题**：压缩/多义是模型能力的来源，模型天然反人类阅读；一切拆解是缓解非根治。
- 11.3 **追不上模型迭代**：移动靶，且靶跑得比子弹快；投入产出比天然吃亏。
- 11.4 **"成功"定义模糊**：缺公认标尺，"理解了一个电路"算不算 progress 难判，努力难转化为成果与"硬"论文。（与第 9 节评估互为表里。）
- 11.5 **规模与算力错配**：有意义的对象（前沿大模型）做不起，做得起的（小模型）意义被质疑。
- 11.6 **劳动密集且"脏"**：人肉看特征、手动标注、反复干预，前沿光鲜底下是大量不确定回报的体力活。
- 11.7 **平衡一句**：坑深是因为底下可能埋着金子——问题极根本（诱人）+ 反馈极稀缺（折磨）两个属性同时存在；上限极高，类比早期神经科学/分子生物学。

# 12. 研究版图
经典派、机制派学术界、独立开源社区、各大公司分别在做什么；澄清"只有某一家在做"的错觉；算力如何塑造学术界 vs. 工业界的分工。

# 13. 开放问题与未来方向
能否 scale 到前沿大模型；superposition 是否被真正解决；自动化瓶颈；忠实性是否可证；与神经科学类比能走多远；阶梯能否从第 3 层稳定推进到第 4、5 层。

# 14. 结论
回扣核心主张与主线：可解释性的本质是"看懂"而非"限制"；忠实性是永恒标尺；理解滞后于能力是这个领域存在的根本理由；其最终价值取决于能在"用处阶梯"上爬多高。
