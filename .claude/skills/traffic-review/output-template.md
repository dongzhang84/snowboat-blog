# 流量复盘输出模板

本文件定义 analysis/*.md 的结构规范。写入时按这个骨架填充。

---

## 文件命名

```
analysis/YYYY-MM-DD 至 MM-DD N篇文章流量复盘.md
```

- 时间段用 ISO 日期（便于排序）
- N 是实际分析的文章数量
- 如果跨月：`analysis/2026-04-28 至 05-02 五篇文章流量复盘.md`

## 格式硬性约束

- 段落 2-4 句，不写长段
- 禁用破折号（——），用逗号、句号、顿号
- 禁 AI 高频词：至关重要、意义重大、标志着、见证、根植于、让我们共同、拥抱、新篇章
- 否定平行句式"不是X，是Y"最多 1 次
- 数字用"3.8k""2000 万"这种可读形式，不用"3800""20,000,000"
- 表格第一列日期用 M/D 短格式（和 README 保持一致）
- 主节用中文数字 + `##`（一、二、三...）

## 写入后要报告

向用户返回：

1. 文件路径
2. 样本文章数
3. 最高流量和最低流量
4. 一条最关键洞察（不超过 50 字）

## 七、逐篇流量预测

**预测方法**：每篇区间依据三个维度。

1. **历史 comp**：从 README 挑最接近的已发文章作对标。
2. **framework 四支柱打分**：身份锁定 / 诊断自救 / 数字反差 / 情绪钩子，各 0/1。
3. **时间修正**：周日 PT 发 = 北京周一早最佳；周四 PT 发 = 北京周五早略弱。

区间下限 ≈ comp × 0.7，上限 ≈ comp × 1.3。

### 章节编号规则

- 复盘正文用到章节几（常见到"五"），下周计划接着往下（从"六"开始）
- 如果复盘只写到"四"，下周计划从"五"开始
- 保证整份文件章节号连续
