# 输出格式

本文件定义文章最终输出的格式规范。

## 文件

- **如果原始素材已经是项目根目录的 md 文件（用户的 outline / 草稿），写回那个文件，不要新建文件**。outline 文件的标题已经是用户给的工作标题，扩写之后的正文直接覆盖原内容，原 outline 内容降级成"附录：原始草稿"的引用块附在末尾
- 只有原始素材在 `private/` 文件夹或直接以对话文本给出时，才在根目录新建一个文件，文件名基于最终的文章标题
- 位置：项目根目录（`/Users/dong/Projects/snowboat-blog/`）
- 这个规则是 2026-05-15 用户明确反馈的：写文章时不要凭空多造一份 md，要把扩写后的正文写回 outline 文件，outline 降级成附录

## 文章末尾必须按顺序包含三节

1. `## 作者其它文章（选）`
2. `## 本文参考文献`
3. `## 附录：原始草稿`

节之间用 `---` 分隔。

### 1. 作者其它文章（选）

列出作者已发表的其它文章，按**从新到旧**排列（最新文章在最上面），用 X 帖子链接。

**不包含当前文章本身**。

数据源：读取项目根目录的 `README.md`，从 "Articles" 表格提取条目。

**筛选规则**：必有 X 链接（硬性前置），再满足以下任一条件即入选（**OR**，不是 AND）：
1. **硬性前置**：跳过 X Post 列为 `—` 或非 `x.com` 链接（如 Medium / Substack）的条目
2. **OR 条件 A — X 流量 ≥ 5k**：X Views 列带"k"且数字 ≥ 5（5.x k、6.x k 等也算；`—`、`< 5k`、`500+`、`4.8k` 等都不算）
3. **OR 条件 B — 最近 10 天**：Date 列在当前日期往回 10 天的窗口内（即使 X Views 是 `—`，最新文章因尚未积累流量也入选）

满足前置 + A/B 任一项即入选。两个条件覆盖两个场景：A 是"作者的历史好文"（让新读者看到过去爆款），B 是"作者最近的产出"（让老读者看到新文章）。

格式：`- [文章标题](X帖子URL)`

示例（假设当前日期 2026-06-02）：

```markdown
## 作者其它文章（选）

- [美国税收制度完全指南](https://x.com/snowboat84/status/2060511915617779821)
- [当物理遇上AI：深度学习里的物理元素（上）](https://x.com/snowboat84/status/2060145538922844179)
- [一文看懂美国的法律系统](https://x.com/snowboat84/status/2059795010330251568)
- [教宗良十四世论人工智能（精华版）](https://x.com/snowboat84/status/2059434342745866391)
- [廉颇老矣，尚能饭否：现代数学史（下）](https://x.com/snowboat84/status/2059071134738620606)
```

**为什么这套规则**：发到 X 时，"作者其它文章（选）"是给读者的延伸阅读列表。太老（>10 天）社交媒体上失去时效性，低流量（<5k）暴露作品冷门。

🔒 **硬性规则**：符合 OR 条件（流量 ≥ 5k 或 10 天内）的**全部入选**。不要按"5 条左右最优"这种主观感觉再砍。规则会自然把条数控制在 5-10 条范围，让规则决定具体条数，作者人工精选的话由作者自己加减，AI 不要主动砍。

📍 **背景**：[2026-06-02] 用户在《人工智能的工程全景（上）》发 X 前明确提出"选 X 流量超过 5k 且最近十天的，没有 X 链接的不要贴"。

📍 **二次踩坑** [2026-06-09]：原版有句"5 条左右最优"作为说明语，被 AI 误读成硬上限。AI 把符合规则的 6-8 条砍到 5 条，删掉了用户精选的历史爆款（移民系统 104k、医疗系统 101k、教宗 18k、税收 16k）。用户明确反问"谁告诉你要精简到 5 条的"。这条删掉所有"5 条左右最优"的暗示性措辞，硬规则就是 OR：流量 ≥ 5k 或 10 天内，符合的全部入选。

### 2. 本文参考文献

- 列出所有引用的来源
- 使用 Markdown 超链接格式：`[标题](URL) - 来源`
- 每条独立一行
- 顺序按正文引用顺序，或按重要性

**注意节名是"本文参考文献"，不是"参考文献"**。

示例：

```markdown
## 本文参考文献

- [Pieter Levels 个人网站](https://levels.io) - 自述
- [NASA Artemis II Mission Overview](https://www.nasa.gov/artemis-ii) - NASA 官方
```

### 3. 附录：原始草稿

- 将用户的原始草稿以引用格式（`>` ）附在最后
- **不要修改原始草稿的任何内容**，包括其中的破折号、错别字、不通顺的表述
- 标题格式：`## 附录：原始草稿`

示例：

```markdown
## 附录：原始草稿

> 这是用户的原始笔记。
>
> 可能包含破折号——这里也不改。
```

## 完成后告知

向用户返回：

1. 文件完整路径
2. 文章结构（主节标题清单）
3. 字数（大致）
4. 本文参考文献条数
5. 作者其它文章（选）条数
