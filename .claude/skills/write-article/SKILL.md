# 整理文章

将用户提供的零散内容整理成一篇结构完整的文章。

## 输入方式

用户可以通过以下方式提供素材：
- 指定一个或多个文件路径：读取文件内容作为素材
- 直接在 `$ARGUMENTS` 中描述主题和要点
- 两者结合

## 整理流程

### 第一步：收集素材

1. 如果 `$ARGUMENTS` 包含文件路径，读取所有指定文件
2. 提取所有关键观点、数据、案例、引用
3. 识别素材中的核心主题

### 第二步：分析与规划

1. 确定文章的核心论点（一句话能概括的中心思想）
2. 将零散内容按逻辑关系分组归类
3. 设计文章大纲：
   - 标题（吸引人、点明主题）
   - 引言（为什么读者应该关心这个话题）
   - 主体（3-5个核心段落，每段一个论点）
   - 结尾（总结、升华、或行动号召）

### 第三步：撰写文章

1. 按大纲组织内容，确保逻辑连贯
2. 补充必要的过渡句，让段落之间自然衔接
3. 保留原始素材中的数据、案例和引用，确保准确性
4. 去除重复内容，合并相似观点

## 写作风格要求

- 语言：中文，简洁有力，避免空话套话
- 风格：参考项目中已有文章的风格——直接、有观点、用数据说话
- 段落：每段聚焦一个观点，不宜过长
- 标题：使用 Markdown 格式（# ## ###）
- 如果素材中有英文内容，翻译整合进中文文章，关键术语可保留英文

## 输出

将整理好的文章写入一个新的 Markdown 文件，文件名基于文章标题。完成后告知用户文件路径，并简要说明文章结构。
