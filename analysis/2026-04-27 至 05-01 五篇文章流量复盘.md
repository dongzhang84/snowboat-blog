## 一、为什么 framework 这两次错得这么离谱

### 1. AI 编程：被低估 333%，因为 framework 没有"实用科普方法论"这一类

预测时按"两万字长文 + 大众痛点 + 偏科普缺挑衅"打 3 分，参考的 comp 是 AI 教育（3.5k）。漏掉的 comp 是上周 4/23 的全网最详细 AI 学习路线图（11k）。

学习路线图、AI 编程、Vibe Reading 三连，平均 15k。它们共享一个公式：标题强 claim（全网最详细 / 两万字科普 / 系统化方法）+ 大众认知痛点（学 AI / 学编程 / 读书）+ 方法论 + 可收藏。这一类在 framework 里被打成"3 分中等"，实际天花板在 10k-22k，比 4 分的 AI 创业反思（套壳 / Vibe Writing 平均 3.6k）高 5 倍。

framework 应该补一条：**实用科普方法论类标题，独立分级，预测下限 5k**。

### 2. Vibe Reading：被低估 900%，因为受众池广度算错了

预测时按"读书话题比写作话题窄"打 -15%。这是错的。

X 平台确实偏创作者群体，但"读书"是**所有知识工作者**的共同痛点（程序员、产品经理、研究员、学生、创业者都要读书），"写作"主要是 X 创作者群体（基数小一档）。我把两个倒着排了。

加上三个加成：
- **"Vibe"系列品牌效应**：4/24 Vibe Writing 拿了 3.6k，读者认识"Vibe" 这个标签了。Vibe Reading 直接吃到品牌延续 buff
- **vertical 切换 bonus**：4/27、4/28、4/29 三天 AI 圈内长文连发，4/30 切到读书 vertical 激活了一批新读者，跨 vertical 切换打破了"日预算耗尽"
- **强 claim 标题**："系统化方法"是信息保证语，跟两万字科普同档

四件事叠加，2.2k 跟 22k 之间隔了 10 倍。

framework 至少要加两条：**audience pool size**、**系列品牌延续**。

### 3. AI 新词：高估 23%，因为没扣"标题专名密度"

四支柱满分（4 分）按规则应该 3-6k，实际 3.1k 卡在区间下限。

问题在标题：**"Prompt、Context、Agent、工程主线"4 个技术专名同时出现**，对非技术读者一秒判断"圈内话题"划走。framework 之前没有"标题专名密度"这一项扣分。

加上前一天 AI 编程 13k 把"愿意读 AI 长文"的读者注意力吃了一波，连续两天 AI 圈内 vertical 又触发了耗尽，3.1k 算合理。

framework 要补一条：**标题出现 ≥3 个技术专名，受众池基数自动压一档（-30% 到 -50%）**。

### 4. Manus：低估 30%，预测合格

预测 2k-4.5k 中位数 3k，实际 3.9k 落在区间内偏上。

"政治敏感首小时算法压制"规律继续生效（首日 2.2k），但靠读者长尾分发慢慢爬到 3.9k。错过 NDRC 叫停热点峰值 3 天的判断也对了，没让流量飞到 6k+。这一条的归因模型是合格的。

### 5. 气吞万里如虎：未预测，2.7k 是合理基线

19 世纪数学家这种窄题材，X 上华语读者基数本就小，加上周五 PT 发卡到 BJ 周六早的周末窗口，双重扣减下 2.7k 属于正常水平。同档前作 SpaceX 立志传一 1.4k 类似。

## 二、framework 需要补的三条新规律

总结四个偏差的根因，framework.md 里需要追加：

1. **实用科普方法论独立分级**：标题强 claim（"全网最详细" / "两万字" / "系统化方法"）+ 大众认知痛点（学习 / 工作流） + 可收藏方法论。预测下限 5k，天花板在 15-22k，跟传统四支柱解耦
2. **audience pool size 入打分**：题材本身的目标受众有多大，独立打 1-3 分。读书 / 编程 / 学习方法 / 职场是高基数；AI 圈内技术 / 政治深度 / 学科历史是低基数
3. **标题专名密度扣减**：标题里 ≥3 个技术专名（Prompt / Context / Agent / Transformer 这种），受众池基数自动压一档

加上之前已经在 pending-notes 验证过的两条（实用清单+强claim上限远高于叙事；政治敏感首小时压制），framework 要补的累计 5 条规律。
