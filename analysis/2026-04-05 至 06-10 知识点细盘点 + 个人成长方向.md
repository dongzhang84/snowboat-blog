## A. AI 工程栈（核心承重资产）

最近 1 个月深度投入的承重线。已发上篇，中篇 7 章扩稿待发，下篇骨架已搭。

### A.1 硬件生态演化（上篇 Ch1-2）
- **CUDA 的"非 AI 投入"豪赌**：NVIDIA 1990s GPU 是游戏显卡 → 2006 CUDA → 2012 AlexNet 转折
- **英伟达赢的真正理由**：不是芯片本身，是 CUDA 生态 + cuDNN + 持续架构迭代
- **TPU 的"专用/极致"路线 vs GPU 的"通用并行"路线**：谷歌 10 年布局
- **第三极硬件**：AMD MI300、AWS Trainium、Cerebras、华为昇腾
- **中国 AI 硬件生态**：管制逼出来的另一套，跟国产软件栈协同
- 文章：《人工智能的工程全景（上）》Ch1-2

### A.2 供应链与网络（上篇 Ch3）
- **三大物理咽喉**：晶圆（TSMC）、HBM（SK 海力士 / 三星）、CoWoS 封装
- **地缘集中度**：CoWoS 几乎全在台湾
- **结构性短缺贯穿 2027**：HBM 供给跟 GPU 销量错位
- **网络分层**：NVLink/NVSwitch（scale-up）vs InfiniBand/Ethernet（scale-out）vs all-reduce 瓶颈 vs NIXL 协议
- 文章：《人工智能的工程全景（上）》Ch3

### A.3 电力（上篇 Ch4）
- **量级直观化**：单机柜 100kW，单设施 1GW，一座大型 AI 数据中心 ≈ 一座中型核电站
- **电网取代芯片成为新瓶颈**：芯片可以买，电网建不起来
- **美国选项**：核电、SMR（小型模块化核反应堆）、地热
- **中美对比**：中国电力充裕但芯片管制，美国芯片自由但电网瓶颈
- **tokens per watt 新指标**：替代纯 FLOPs
- 文章：《人工智能的工程全景（上）》Ch4

### A.4 训练态（上篇 Ch5）
- **训练经济 anchor**：SOTA 模型一次训练几千万到几亿美元
- **4D 并行**：Data + Tensor + Pipeline + Sequence
- **ZeRO/FSDP 显存分片**：把单卡装不下的模型分片到多卡
- **Fault tolerance**：万卡集群一定会坏，checkpoint 怎么做
- **数据 pipeline 被低估**：训练时数据 I/O 占总时间不小
- 文章：《人工智能的工程全景（上）》Ch5

### A.5 推理基础（上篇 Ch6-7）
- **prefill / decode 两阶段**：prefill 算力密集、decode 显存带宽瓶颈
- **KV cache 机制**：避免重复计算注意力状态
- **PagedAttention**：把显存当虚拟内存管，vLLM 起源
- **Continuous batching**：动态填充 GPU
- **PD 分离**：prefill 跟 decode 拆到不同硬件配置
- 文章：《人工智能的工程全景（上）》Ch6-7

### A.6 推理优化深度（中篇 Ch1）
- **量化**：FP16 → FP8 → INT4/FP4 阶梯；GPTQ、AWQ、SmoothQuant 三大算法；FP4 是 Blackwell 后主流
- **蒸馏的 Teacher-Student 范式**：Hinton 2015 起源；学的不是 token 序列而是 teacher 的概率分布；GPT-4 → GPT-4o-mini、Claude Opus → Haiku
- **剪枝**：非结构化（置零）vs 结构化（删整层）；单独效果差，要跟蒸馏量化叠加
- **MoE**：Mixtral 8x7B、DeepSeek V3 671B 总参数 37B 激活；专家显存装载是工程难点
- **投机解码三代**：草稿-目标双模型 → Medusa（解码头）→ EAGLE
- **推理引擎**：vLLM（开源标准）、TensorRT-LLM（性能最高）、SGLang（共享前缀）、lmdeploy（国内）
- **推理三角权衡**：成本/延迟/吞吐不可兼得
- **测试时计算**：o1、R1 把推理时算力当 scaling axis
- 文章：《工程全景中篇》Ch1

### A.7 后训练（中篇 Ch2-5）
- **为什么需要后训练**：base model 只会续写不会对话；GPT-3 → ChatGPT 转折秘密是 InstructGPT
- **SFT 监督微调**：1.3B InstructGPT 偏好打分优于 175B GPT-3
- **RLHF 三步**：SFT → reward model → PPO
- **Reward hacking 与 sycophancy**：reward model 偏差被 RL 放大
- **对齐税**：HHH 三者内在冲突
- **DPO**（Rafailov 2023）：把 RL 拿掉的洞见，policy 就是 reward model
- **DPO 变体**：IPO / KTO / ORPO
- **Constitutional AI**（Anthropic 2022）：用宪法让 AI 自我批评
- **RLAIF**（Google 2023）：AI 反馈替代人类
- **蒸馏式后训练**：Vicuna、Alpaca 走的路
- **Model Collapse**（Shumailov 2024 Nature）：合成数据反复训练分布坍塌
- 文章：《工程全景中篇》Ch2-5

### A.8 对齐与安全（中篇 Ch6-7）
- **HHH 框架**（Askell 2021）：Helpful / Honest / Harmless
- **Helpful vs Harmless 钢丝**：过度拒绝 vs 过度配合
- **Honest 的四个层次**：知道自己不知道、不编引用、不附和错误、calibrate confidence
- **Scalable Oversight**：当模型超过人类水平怎么对齐
- **OpenAI Superalignment 团队 2024-05 解散**：Ilya + Leike 出走
- **Jailbreak vs Prompt injection**：前者绕 refusal 边界，后者污染指令信任
- **红队（Red Teaming）**：内部对抗测试
- **欧盟 AI Act 分阶段**：2024-08 生效，2025-02 禁止条款、2025-08 GPAI、2027-08 全覆盖
- **加州 SB 53**：2025 通过，2026 实施
- **可解释性研究**：Anthropic Scaling Monosemanticity（3400 万 features）、Golden Gate Claude、circuit tracing、introspection、Dario 2027 目标
- 文章：《工程全景中篇》Ch6-7

## B. LLM 商业判断

独立于工程栈的判断框架，最近发的《大语言模型的商业痛点》是代表作。

### B.1 四象限判别尺子
- **Beta（可迭代）**：会被下一版模型抹平，押它等于押大厂失败
- **根本（清不零）**：范式自带的天花板，加数据加参数也好不了
- **Alpha-技（外部补缺口）**：模型从内部修不了，第三方层兜底
- **Alpha-构（结构性壁垒）**：大厂结构性去不了的场景
- 文章：《大语言模型的商业痛点》Ch2

### B.2 18 个具体痛点的分类
- 幻觉（量级 Beta、地板根本、围绕地板做 Alpha-技）
- 非确定性（半根本）
- 长链逻辑（可验证域 Beta、不可验证域根本）
- 长上下文与跨会话记忆（窗口 Beta、深度记忆 Alpha-技）
- 啰嗦谄媚（Beta 但治标）
- 成本与延迟（Beta，大厂硬件碾压式下降）
- prompt 脆弱（context engineering 取代）
- 版本退化（Alpha 监测层）
- 评估难（元痛点，最硬 Alpha）
- 安全 prompt injection（Alpha-技）
- 合规与责任（Alpha-构，Harvey / Hippocratic AI 模式）
- 黑盒不可调试（Alpha + 部分根本）
- 法律（版权 / 隐私 / unlearn）
- 厂商锁定与模型下线（Alpha-构）
- 文章：《大语言模型的商业痛点》Ch3-5

### B.3 学术指标 vs 工业指标的鸿沟
- benchmark 跑分跟"敢部署到生产"是两件事
- **OpenAI 2025-09《Why Language Models Hallucinate》**：证明幻觉是预训练统计目标的范式级问题
- 文章：《大语言模型的商业痛点》Ch1

### B.4 宏观判断
- **capex 循环融资**：AI 公司拿钱买云算力，云厂算成收入，估值上去
- **Scaling wall**：Ilya NeurIPS 2024 + Villalobos 数据墙（公开文本 2026-2032 耗尽）
- **应对方法**：RL / reasoning、合成数据、小专家舰队、世界模型
- 文章：《大语言模型的商业痛点》Ch6
