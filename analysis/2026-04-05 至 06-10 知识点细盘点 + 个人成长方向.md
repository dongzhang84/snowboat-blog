# 第一部分：已经讲过的具体知识点

## A. AI 工程栈

最近 1 个月深度参与的承重线。已发上篇，中篇待发，下篇骨架已搭。覆盖的核心概念：

- **硬件演化主线**：CUDA 2006 的非 AI 投入豪赌、AlexNet 2012 转折、谷歌 TPU 10 年布局、英伟达赢的真正理由（CUDA 生态 + cuDNN + 持续架构迭代，而不是芯片本身）
- **供应链三大物理咽喉**：晶圆（TSMC）、HBM（SK海力士/三星）、CoWoS 封装（几乎全在台湾）。地缘集中度是 2027 前的结构性短缺源
- **电力新瓶颈**：单机柜 100kW、单设施 1GW 量级。tokens per watt 取代纯 FLOPs 成为新效率指标
- **训练态工程**：4D 并行（DP+TP+PP+SP）、ZeRO/FSDP 显存分片、Fault tolerance 处理万卡集群必坏的现实
- **推理两阶段**：prefill / decode、KV cache 机制、PagedAttention（vLLM 起源）、PD 分离
- **模型瘦身四条**：量化（GPTQ / AWQ / SmoothQuant / FP4）、蒸馏（Teacher-Student，Hinton 2015）、剪枝、MoE（Mixtral、DeepSeek V3 671B/37B）
- **投机解码三代**：草稿-目标双模型 → Medusa → EAGLE
- **后训练**：SFT、RLHF 三步、DPO 把 RL 拿掉的洞见、Constitutional AI、RLAIF、Model Collapse（Shumailov 2024 Nature）
- **对齐与安全**：HHH（Askell 2021）、对齐税、Jailbreak vs Prompt injection、欧盟 AI Act 分阶段、可解释性（Scaling Monosemanticity、Anthropic 2027 目标）

承重砖密度最高的一条线，但**篇数远超其他 topic**，下一步是收尾整合而不是再加散文。

## B. LLM 商业判断

最近发的《大语言模型的商业痛点》是这条线的代表，独立于工程栈之外。核心 framework：

- **Beta / 根本 / Alpha-技 / Alpha-构 四象限判别尺子**：哪些痛点会被下一版模型抹平，哪些不会
- **18 个具体痛点的四象限分类**：幻觉、非确定性、长链逻辑、长上下文、啰嗦谄媚、成本、prompt 脆弱、版本退化、评估难、安全、合规、黑盒、版权、厂商锁定等
- **学术指标 vs 工业指标的鸿沟**：benchmark 跑分跟"敢部署到生产"是两件事
- **评估难是元痛点**：放大其他一切缺陷的根本障碍
- **Sycophancy 是 RLHF reward hacking 的产品化体现**
- **版本退化与厂商更新破坏 pipeline**
- **宏观判断**：capex 循环融资 vs ROI、scaling wall（Ilya NeurIPS 2024 + Villalobos 数据墙）

这是承重砖里**独立成体系的判断框架**，跟工程栈互补。读者读完能拿到一把可调用的尺子。

## C. AI 工具方法论（Vibe X 系列）

你自己原创的术语族。4 篇文章建立了一套"AI 时代的知识工作方法论"。

- **Vibe Reading**：AI 时代读书的系统化方法。不再是"读完一本书"而是"用 AI 把信息流转化成可调用的判断"。你定义了"读 → 摘 → 提问 → 复用"的循环
- **Vibe Writing**：AI 时代写作的工程化。提纲驱动 + 反复迭代 + skill 系统化。你自己的写作流程是这套方法的活样本
- **Vibe Learning**：AI 时代学习被重新组织。学习从"系统化教程 → 知识点"变成"问题驱动 + 即时获取 + 内化框架"
- **Claude Skill 三个最有用的**：把抽象方法论落到具体工具上
- **Prompt → Context → Harness 三层工程范式**（《别再被 AI 新词绕晕了》）：这是承重砖中的承重砖。Karpathy 2025-06 的官方背书，但你 2025-04 已经写过。三层范式直接进了后来的《商业痛点》4.2 节、工程全景中篇 Agent 那一面

这一线是你**真正独立发明的概念族**，跟搬运式整理完全不同。Vibe X 这个词族在中文 AI 圈已经有一定辨识度。

## D. AI 工程主线 / 概念辨析

4 篇深度长文，每篇都是一个核心概念的系统讲解：

- **《一篇文章讲清大语言模型发展史》**：从 word2vec → seq2seq → attention → Transformer → BERT → GPT 系列 → 现代 LLM 的完整脉络。配上每个时代的代表论文和工程突破
- **《两万字科普：AI 为什么会编程》**：原理 + 历史 + 未来。覆盖 Copilot、Cursor、Claude Code 的工程演化，以及"AI 写代码"为什么不只是 prompt 工程
- **《什么是 Diffusion Model》**：图片视频生成模型的完整原理介绍。覆盖 DDPM、score-based、latent diffusion、Stable Diffusion 到 Sora 的演化
- **《什么是世界模型？一个正在被争夺的概念》**：2024-2026 年世界模型的概念战争。Yann LeCun JEPA、李飞飞 World Labs、Sora 视频生成派之间的路线之争
- **《什么是控制论？控制论是 AI 的上辈子吗？》**：把控制论作为 AI 早期思想史接上。Wiener 的反馈、贝特森的系统论、跟现代 RL 的关系

这一线建立的是"AI 概念辨析能力"：每篇都把一个被业内含糊用的词彻底拆清楚。是承重砖。

## E. 数学史 + 数学物理交叉

5 篇，覆盖三个时代的数学发展。

- **《气吞万里如虎：十九世纪的数学英豪们》**：高斯、黎曼、伽罗瓦、康托尔。数学第一次大爆发的时代背景：欧洲学院制度兴起 + 拿破仑数学化教育 + 哥廷根学派的传承
- **《我见青山多妩媚：二十世纪数学史（上）》**：希尔伯特纲领、哥德尔不完备、布尔巴基学派的形式化运动
- **《廉颇老矣，尚能饭否：现代数学史（下）》**：20 世纪后半叶的数学—— Atiyah-Singer、Wiles 证明费马、Perelman 证明庞加莱。也讲了"现代数学过于形式化、跟物理实验脱节"的内部反思
- **《祖父积分学概论》**：教法变迁的故事。从牛顿莱布尼茨原始的力学/几何直觉、到柯西的极限严格化、到现代分析的教学法
- **《卡拉比丘成桐定理及其物理意义》上中下**：从复几何到弦理论。讲清楚为什么纯数学（卡拉比猜想、丘成桐证明）会成为物理理论（弦论的卡拉比丘流形）的基础

这一线建立的是**"用人物史 + 时代背景讲数学"的写作模式**，比纯技术教科书更易读。你的天体物理 PhD 背景让这些写作有可信度。

## F. AI + 物理学交叉

你的核心独特资产线，跟 PhD 出身直接对接。目前 3-4 篇，还有巨大空间。

- **《学物理的兄弟们，物理学已死，请转行搞 AI》**：劳动力转型论。物理 PhD 应该把 AI 当作新的物理来做，不应该死守传统物理。讲了几个具体的迁移路径
- **《当物理遇上 AI（上）：深度学习里的物理元素》**：Scaling Law（Kaplan 2020、Chinchilla 2022）是物理学家熟悉的标度关系。双下降现象（Belkin 2019）跟相变类比。随机矩阵理论（Marchenko-Pastur 分布）解释神经网络泛化
- **《当物理遇上 AI（下）》**：临界现象在大模型里的回响。涌现是不是相变（争论中）
- **《什么是控制论》**：把控制论作为 AI 的"上辈子"，接上了 Wiener、Shannon、Ashby 这条早期思想史。物理学家在 20 世纪给 AI 打的地基

这一线的写作有一个明显特征：**用物理学训练出来的"找守恒量、找标度、找对称性"思维看 AI**。这是别人模仿不来的。

## 整体节奏建议

每周发 4-5 篇的节奏可以适度放缓，转向"重质不重量"：

- 每周 1 篇承重砖（工程栈 / AI 物理交叉 / 商业判断）
- 每周 1-2 篇借势砖或独立兴趣（华人观察 / 数学物理 / 概念辨析）
- 不再追"美国全生态"或"AI 政策散文"的快产出

预期流量从最近两周的 130k 量级回到 60-80k 量级，但承重砖比例从过去 30% 拉到 60%+。

3-5 年后再回头看，承重砖才是真正的写作护城河。
