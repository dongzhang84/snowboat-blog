# 引子：一个中文词，两件相反的事

> 题眼钩子。用术语撞车破题，一句话立住全篇的张力。

- 0.1 中文"推理"塌了两个英文词：**inference** 和 **reasoning**
- 0.2 两者其实方向相反：inference = 把训好的模型跑一遍（工程/执行）；reasoning = 答题前的思考（能力/范式）
- 0.3 词源考据：inference 本是统计学的"推断"，简中自媒体误译成"推理"才造成撞车
- 0.4 最妙的反转——reasoning 并没跳出 inference，它**寄生**在 inference 上（test-time compute）：用"放大推断"来"买推理"
- 0.5 全文术语约定：本文 reasoning 一律写"推理/思考"，inference 一律写"推断/运行"

# 一、推理不是新东西，是 AI 最老的梦

> 反直觉的历史铺垫：推理先死过一次，才在 LLM 上重新长出来。

- 1.1 符号推理时代（1950s–1980s）：Logic Theorist、GPS、专家系统、Prolog、定理证明——最早的 AI 几乎全是推理机
- 1.2 那种推理"像几何"：从公理/规则出发，if-then 一步步演绎，每步可验证
- 1.3 撞墙与寒冬：规则脆、知识录不进、处理不了模糊、组合爆炸
- 1.4 深度学习翻盘：从"对符号做推理"翻成"对数据找规律"，推理被挤成涌现的副产品
- 1.5 一句对照——**老 AI 全是推理、零学习；现代 AI 全是学习，推理得重新长**

# 二、思维链（CoT）：推理从"提示"里冒头

> 回答"推理是不是 prompt engineering 之后才有的"。

- 2.1 CoT 是什么：先写步骤、再给答案（火车时速题做例子）
- 2.2 它最早是一个 **prompt engineering 技巧**：零样本"一步步想"、少样本给带步骤的范例
- 2.3 配套招式：self-consistency（多条链投票）、Tree of Thoughts
- 2.4 关键洞见：推理能力本就潜伏在预训练参数里，prompt 只是把它**勾**出来的钥匙
- 2.5 天花板：靠提示（忘了就缩回）、不知道自己错、没法稳定加长、只能榨已有的能力
- 2.6 结论——与其每次提示它想，不如把"想"**训进**它的本能

# 三、推理到底是什么：草稿纸上的搜索

> 全篇的概念核心。把"思考"祛魅，回答"它到底在想什么"。

- 3.1 地基：模型其实只会一个动作——**看着前面的字，猜下一个字**
- 3.2 "思考"= 它在正式答题前，给自己**多吐的那一大段草稿**（草稿本身也只是猜字）
- 3.3 为什么有用：把"一次大跳跃"拆成"很多次小跳跃"，每写一句给下一句铺垫脚石
- 3.4 三个比喻给推理定性：
  - 不像几何（没有保真的逻辑引擎——那恰是被抛弃的老 AI）
  - 不是天马行空（预训练强先验 + RL 奖励约束着它）
  - 最准：**在草稿纸上做搜索**——提一步、检查、回溯、再试
- 3.5 "想久一点"到底在干嘛：草稿更长 = 多铺步 + 多试路 + 多检查；"aha moment"= 多写一句"等等，错了"再改
- 3.6 收口——reasoning 说穿了就是"答题前那段一步步铺垫、试错、检查的草稿"，没有更玄的东西

# 四、训练全景：能力在哪一段长出来

> 拆掉最大的认知误区，建立位置感。

- 4.1 先拆名字误会：**"training" 是整个过程的统称，不是中间的独立步骤**；真实结构只有 pre-training → post-training 两大段
- 4.2 参数 = 几百亿个旋钮；训练 = 拧旋钮；reasoning 是**拧出来的习惯**，不是加进去的零件（参数数量不变，只是数值变）
- 4.3 pre-training（主训练，最贵最久）：海量文本"猜下一个字"，把语言规律 + 世界知识灌进**全部参数** → 得到博学但不听话的 base model
- 4.4 post-training（后训练，轻装修）：在博学地基上**轻微调同一批参数**，教对话、教推理
- 4.5 为什么 post 不会把 pre 调乱：① 只轻拧 ② 主要动"管行为"的旋钮而非"存知识"的旋钮 ③ KL 惩罚这根绳子 ④ 灾难性遗忘是真问题但有成熟对治
- 4.6 区分三种状态：训练中（旋钮在动）vs 使用时 / inference（旋钮冻死、只读）
- 4.7 反直觉结论——**不重练昂贵地基，只在后段一拧，推理就出来了**（R1 的震撼正在此）

# 五、发动机：用 RL 把推理训进去

> 全篇最硬的一章。每个术语先配大白话锚。

- 5.1 为什么必须用 RL：推理没有"字帖"（同一题一百种对的草稿），不能临摹，只能"给分数、自己琢磨"
- 5.2 两种 RL 千万别混：
  - RLHF / RLAIF：管"礼貌、有用、无害"，靠人打分 / AI 照宪法打分——老东西，**没壁垒**
  - RLVR（可验证奖励）：管"把题做对"，靠机器自动判分——新东西，**这才是推理的发动机**
- 5.3 四个零件（用火车题对一遍）：策略（模型本身）/ 动作（吐的字）/ 轨迹（整段草稿+答案）/ 奖励（最后对没对）；稀疏奖励的难处
- 5.4 核心机制——策略梯度：把"高分轨迹的那些选择"的概率往上顶，低分往下压
- 5.5 关键升级——优势（advantage）：不看绝对分，看"比我平均水平好多少"
- 5.6 两条工程路线（重点对比，最有信息量）：
  - PPO：再养一个 critic 网络去预测"平均水平"，重、贵、难调
  - GRPO：**砍掉 critic**，同题采一组答案、组内平均当基准——R1 能廉价复现并开源引爆的关键 trick
- 5.7 奖励设计（RLVR 精髓与陷阱）：数学对答案 / 代码跑测试 / 格式分；reward hacking（钻空子蒙分）与堵空子的攻防
- 5.8 防练疯——KL 惩罚：允许进步，不许为刷分偏离原模型太远
- 5.9 训练循环五步：采样 → 打分 → 算优势 → 更新策略 → 循环
- 5.10 深层意义——模型拿自己生成的草稿训自己（自我对弈、数据自给），绕开"人类数据用完"的墙

# 六、产品史：从 o1 到 R1 到今天

> 把抽象机制落到真实模型上，时间线清晰。

- 6.1 o1（2024.9）：把长 CoT 从"提示"搬进"训练"内化；**test-time compute 成第三条 scaling 轴**；隐藏原始思维链
- 6.2 DeepSeek R1（2025.1）四个震撼：
  - R1-Zero：纯 RL、跳过 SFT，证明推理能从零被逼出来
  - GRPO：砍 critic 省成本
  - "aha moment"：自发学会回溯（印证第三章"搜索而非演绎"）
  - 蒸馏进小模型 + 开源论文 → 引爆全行业
- 6.3 收尾打磨：冷启动 SFT + 多阶段 RL，修掉语言混杂、思考难读
- 6.4 2025–2026 格局：o3 / o4-mini / o3-pro；四种形态（全隐藏 o3、可查看 Claude extended thinking、流式 R1、并行 Gemini Deep Think）；GPT-5 用路由模型按难度选档；开源 Kimi 等跟进
- 6.5 大图——三条 scaling law 并列（预训练 / 后训练 RL / test-time），2026 重心转向"效率 scaling"

# 七、成本与算力：钱往哪烧，能力就在哪

> 纠正"后训练很便宜"的旧常识，用成本结构讲范式转移。

- 7.1 芯片真相：pre 和 post 用**同类 GPU**，差别只在用多少、用多久；inference 阶段可用更省的卡
- 7.2 旧常识被打破：reasoning 的 RL 要海量自生成长草稿、跑几十万到几百万轮，成本已逼近 pre-training
- 7.3 "post-training 是零头"→"pre 与 post 同一量级"的转变
- 7.4 钩子——**成本结构的迁移，本身就是范式转移的证据**：能力来源从 pre 移向 post 的 RL

# 八、还没解决的问题

> 综述该有的批判性收尾，也为后续文章挖坑。

- 8.1 忠实性（faithfulness）：它嘴上说的推理 ≠ 它真实的内部计算（直通"可解释性"专篇）
- 8.2 想太多 / 想太少：over-thinking 与 under-thinking，置信度校准失灵
- 8.3 reward hacking 的长期军备竞赛
- 8.4 边际递减："想得更久"还能走多远；效率 scaling 是不是新瓶颈
- 8.5 下一步：RLVR 从数学/代码扩到 **Agent / 工具使用 / 网页操作**（RL 与 Agent 即将合流）

# 结语：推理是"拧"出来的习惯

- 9.1 一句话复述定义：reasoning = 用 RLVR 拧进参数的"答题前先打有用草稿"的习惯
- 9.2 三个最易混概念归位：参数（训练时拧、使用时冻）/ inference（运行时猜字）/ reasoning（运行时多吐的草稿）
