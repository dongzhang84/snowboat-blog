# 第一章 硬件生态的缘起

> 主题：不从"现状"切入，而从"历史"切入。先讲清楚今天的格局是十几年里一系列选择的复利结果，读者才能理解为什么是英伟达、为什么 TPU 现在回来了。

- **史前：GPU 本是游戏显卡**。GPU 含数千个小核心、天生擅长并行的矩阵乘，而这正是神经网络的核心运算——但 2000 年代初没人为 AI 设计它
- **2006：CUDA 这一步豪赌**。英伟达把 GPU 从"渲染游戏画面"改造成"通用并行处理器"，黄仁勋顶着 2008 金融危机的股价压力坚持了整整十年 *[研发投入数字写正文前需 fact-check 公开年报]*
- **谷歌 TPU 的另一条路**。谷歌很早为自己的推荐系统和 TensorFlow 设计专用张量芯片（TPU），走的是"为单一目的做到极致"的封闭路线——这是理解今天 TPU 回归的伏笔
- **2012 转折点：AlexNet**。多伦多团队仅用两张普通 GTX 580 游戏卡训练神经网络，把 ImageNet top-5 错误率从约 26% 砍到 15.3%，甩开第二名 10 个百分点以上——证明 GPU 不只是游戏部件，而是下一波计算的引擎 *[具体百分点写正文前需 fact-check 原 paper / ImageNet 2012 leaderboard]*
- **为什么是英伟达赢、而非别人**：真正的护城河不是芯片，是 CUDA 软件生态——它把开发者"锁"了进来。硬件（Tensor Core）+ 生态（CUDA）+ 互联（NVLink）三位一体
- **现状定位**：英伟达约 92% 的 AI GPU 份额、70%+ 毛利——"软件护城河十年内无人能复制"的事实垄断 *[份额和毛利数字写正文前需 fact-check 最新季报口径]*

# 第二章 格局松动

> 主题：承接第一章的"垄断"，讲清楚 2025–2026 真正的结构性变化——不是英伟达倒下，而是第一次出现了有规模的替代力量，撬动它的杠杆是"推理"。

- **核心事件：TPU 的规模化回归**
    - Anthropic 签下谷歌史上最大 TPU 订单（2026 数十万颗 → 2027 约百万颗）
    - Gemini 3（被视为 SOTA 前沿）全程在 TPU 上训练——证明 TPU 已通过"预训练前沿模型"这个最硬的考验
    - 2026 年 4 月起谷歌对外出售 TPU，十年来第一次
- **第三极：AWS Trainium**，以及微软 Maia、OpenAI 自研芯片等"自研加速器"浪潮
- **关键判断：不是替代，是多平台并存、按负载选芯片**
    - 标本：Anthropic 三平台并行（TPU + Trainium + 英伟达 GPU），不依赖单一供应商，每种负载放最合适的芯片上
- **撬动垄断的杠杆，为什么是推理？**
    - 训练焦点是"堆昂贵算力"；推理焦点是"降低每 token 成本以经济地规模化"
    - 在后者上，专用芯片性价比优势凸显（TPU v6e 对 H100 约 4× 性价比）
    - 一句话：推理经济学正在松动英伟达靠训练建立的垄断
- **中国线：被管制逼出来的另一套生态**（可作为对照）
    - 出口管制反而加速国产替代；华为约 62% 国内份额；DeepSeek 证明了低成本推理路线的可行性

# 第三章 供应链与网络

> 主题：芯片只是终点，真正决定能造出多少算力的是上游卡点 + 把上千个加速器连起来的网络。这一章揭示"算力短缺"在物理上到底卡在哪。

**1. 三大卡点，缺一不可**

- 先进制程逻辑晶圆（2nm/3nm）+ 高带宽内存（HBM）+ 先进封装（CoWoS），任何一个短缺都会限制前沿算力的扩张速度
- **CoWoS 封装**：几乎所有高端 AI 加速器都靠台积电的 CoWoS，把逻辑芯片和 HBM 堆叠集成到一块基板上；前四大设计商（英伟达等）吃掉约 80–85% 的产能
- **HBM 内存**：由 SK 海力士、三星、美光三家供应；2026 年总需求约 320 亿 Gb，英伟达独占约 58%；HBM4 进入早期送样，量产要等 2026 末到 2027
- **产能现实**：台积电 2nm/3nm 已订满到 2027+，CoWoS 交期拉到 50+ 周；台积电 CEO 直言要到 2027 年"供给才能追上需求"
- **结构性而非周期性**：过去芯片短缺是周期波动，AI 这轮是"架构性短缺"——需求曲线结构性高于产能曲线，贯穿到 2027
- **地缘集中度**：全球被订满/严重受限的关键晶圆厂几乎全在中国台湾（10 家级别）和美国（2 家）——一个极度集中的物理咽喉
- **外溢效应**：出口管制下英伟达把台积电产能从对华 H200 转向下一代 Vera Rubin（美国客户确定订单）——供应链已被地缘政治直接重塑

**2. 节点之间：网络才是分布式训练/推理的真瓶颈**

- 单卡再强，跨卡通信慢，万卡集群也跑不起来。这是大模型时代被外界严重低估的工程层
- **节点内（scale-up）**：NVLink/NVSwitch 把 8 卡或 72 卡（GB200 NVL72）连成"准单卡"，带宽达 1.8 TB/s 量级
- **节点间（scale-out）**：InfiniBand（NVIDIA 系，延迟低）vs RoCE/Ethernet（开放生态，云厂商更偏好）的路线之争。AWS / Google / Meta 都在重投以太网，赌它最终能追平 IB
- **All-reduce 通信瓶颈**：训练时梯度同步占 30–60% 时间，新算法（ZeRO++、gradient compression、ring/tree all-reduce）就是为压这一项
- **推理跨 GPU 传 KV cache**：第七章 PD 分离要求节点间高速传 cache，NIXL 这类标准化层的出现就是为这件事
- **判断**：网络成本和复杂度在万卡级集群里已超过单卡算力本身的占比

# 第四章 电力

> 主题：基础层最被低估的一章。瓶颈已经从硅片转移到电网。重点做中美对比与国际分布。

- **量级直观化**：单个大型设施需 100 MW–1 GW（相当于 8 万–80 万户家庭）；单机柜 100–140 kW，逼得液冷成为强制项
- **"power beat compute"**：拿到电、地、冷却、并网，现在比算法创新构成更硬的护城河；大量已宣布的数据中心产能卡在"并网难、电网设备短缺"上，迟迟不能落地
- **美国现状**：数据中心约占全国用电 4%，2030 年预测升至 8–12%；各区电价相差 4 倍（$0.04–$0.16/kWh），选址成最大变量成本；弗吉尼亚、得州、佐治亚等集群周边居民电价已上涨 8–15%
- **中美对比（本章核心）**
    - 美国：老化电网是主要瓶颈，审批与并网慢；转向"表后电力"——核电协议、SMR（小型模块化反应堆）、地热；微软/谷歌/亚马逊集体签核电
    - 中国：发电与电网扩张速度优势、电价结构差异；待补充具体数据对比（发电总量、新增装机、电价、绿电占比）
    - 判断角度：电力可能成为中美 AI 竞赛里比芯片更难绕过的长期变量
- **国际分布**：主权 AI 兴起——各国自建本地化基础设施；中东（廉价能源）、北欧（冷却 + 绿电）等新兴算力地理
- **新指标登场**：tokens per watt（每瓦 token）取代纯 FLOPs——它同时衡量基础设施效率和工作负载优化，是连接"基础层"与"推理层"的桥

# 第五章 训练态：万卡集群的工程

> 主题：基础层讲完之后，先讲怎么用这些基础打仗。训练是 AI 工程里最资本密集、最难做对的一态。讲清楚这一章，"推理为什么是另一种工程"才有对照。

**1. 训练态的经济 anchor**

- GPT-4 量级单次训练公开估算 $80M–$200M（OpenAI 自己披露过区间，加 fault tolerance 重训成本）
- Llama 3 405B 用了约 1.6 万张 H100，连跑数月
- Grok 3 / Gemini Ultra 训练集群规模都在 10 万 GPU 量级
- 这些数字解释了为什么训练态的玩家从几十家迅速缩到 5–8 家：玩不起

**2. 分布式训练的并行策略（four-way parallelism）**

- 数据并行（Data Parallel, DP）：每张卡有完整模型，处理不同 batch；最简单但模型必须能放下
- 张量并行（Tensor Parallel, TP）：把每个矩阵乘按行/列切到多卡，跨卡通信极重，所以只在 NVLink 节点内做
- 流水线并行（Pipeline Parallel, PP）：把模型按层切到不同卡，流水线方式跑 micro-batch；引入"气泡"问题
- 序列并行（Sequence Parallel, SP）：把长序列切到不同卡，2024 后期长上下文场景兴起
- **判断**：现代大模型训练都是 4D 并行（DP+TP+PP+SP）混合，调度极难，是 Megatron-LM、DeepSpeed、ColossalAI 这些框架真正的工程价值

**3. 显存压力与 ZeRO/FSDP**

- 训练显存 = 权重 + 梯度 + optimizer states + activations，optimizer states 占大头（Adam 是权重的 4–8 倍）
- ZeRO（DeepSpeed）三阶段：把 optimizer / 梯度 / 权重分到不同卡，按需 gather
- FSDP（PyTorch 原生 ZeRO 等价物）：facebook 实现，更易用
- 混合精度（BF16/FP8）训练：训练自身也走向低精度，Blackwell 提供 FP8 训练硬件加速

**4. Fault tolerance 与 checkpoint 工程**

- 万卡集群每天有节点故障，单次训练几周不能从零重来
- Checkpoint：周期性把训练状态写到磁盘（每隔几小时），故障后从最近 checkpoint 续训
- Async checkpointing：写盘不阻塞训练；GPU 显存 → CPU 内存 → 磁盘三级 staging
- 弹性训练（torchelastic）：节点掉了自动重组拓扑接着跑

**5. 训练态的"另一半"：数据 pipeline**

- 万卡 GPU 等不起慢数据。数据读取/预处理 throughput 必须跟上训练速度
- 分布式数据集（webdataset、Mosaic streaming）、tokenization 离线化、shard 平衡
- 数据质量比数据量重要：DeepSeek 在数据 curation 上的投入被证明是性能差距的关键

**6. 训练态 vs 推理态：完全不同的工程问题**

- 训练：一次性资本开支，几周到几个月，目标是 throughput（每秒训多少 token）
- 推理：持续运营成本，永远在跑，目标同时含 throughput（成本）和 latency（用户体验）
- 用同一套工程逻辑做两件事会两头不讨好——这是为什么训练框架（Megatron）和推理引擎（vLLM）是两套独立栈

# 第六章 推理层导论

> 主题：进入推理层（第六至第十一章）。这一章只讲两件事——为什么"运行"成了中心，以及推理为什么是一个根本不同的工程问题。具体技术细节（prefill/decode、KV cache）都让给后面专章。

**1. 重心迁移的事实**

- 经济占比：推理占 AI 算力比例从 2025 年约半数，升到 2026 年约三分之二
- 成本曲线：每百万 token 价格三年降约 92%——强调这是"相变"而非渐进改善
- "推理经济"成为独立板块：定制 ASIC、专门的推理服务商（Together、Fireworks 等）崛起

**2. 推理的衡量指标体系**

- 首 token 延迟（TTFT）、token 间延迟（TPOT）、吞吐（throughput）、每 token 成本
- 不同应用对这几个指标的权重不同：对话敏感 TTFT，批处理只看吞吐和成本

**3. 推理的两个阶段（铺垫，详情在第七章）**

- Prefill（预填充）：一次性并行处理整个输入 prompt，算力密集（compute-bound）
- Decode（解码）：逐个 token 自回归生成，受显存带宽限制（memory-bound）
- 关键洞察：两个阶段的瓶颈完全不同，这是后面整套推理优化的起点

# 第七章 显存与吞吐

> 主题：推理的核心矛盾是显存，不是算力。这一章讲怎么把昂贵的显存用到极致——这是 vLLM 之所以能比朴素实现快 3–5 倍的根本原因。

**1. KV cache：推理省算力的根本机制**

- 是什么：把已算过的 token 的 key/value 状态缓存下来，避免每生成一个新 token 都重算整个历史
- 代价：它吃显存，且随上下文长度线性增长——长上下文场景下 KV cache 比模型权重还占地方
- 经验法则：通常预留 GPU 显存的 40–60% 给 KV cache

**2. PagedAttention：把显存当操作系统虚拟内存来管**

- 问题：传统做法给每个请求预分配"最大上下文长度"的连续显存，实测浪费高达 60–80%
- 解法：把 KV cache 切成固定大小的小块（默认 16 token）按需分配，逻辑连续、物理不连续，用块表映射
- 效果：近乎零浪费，同样显存能塞 2–4 倍的并发；由 vLLM 在 2023 年首创，现已是所有主流引擎的标配

**3. KV cache 进一步压缩（2026 的三条路线）**

- 低精度：把 cache 量化到 FP8/INT4，显存减半而精度损失 <1%（Hopper/Blackwell 有原生硬件支持）
- 语义级压缩：在语义块层面压缩而非单 token，激进压缩比下仍保结构
- 重要性驱逐（PagedEviction）：按注意力分数丢弃低重要性 token，显存随驱逐率线性下降
- 分层缓存：LMCache 那类把 cache 在 GPU/CPU/存储/网络间调度的系统；显存不够时下放到 CPU 内存

**4. 批处理：把 GPU 喂饱**

- 静态批处理的病：朴素静态批让平均 60% 的 GPU 闲置（短请求等长请求，槽位被锁死，即"队头阻塞"）
- Continuous batching（连续批处理）：在迭代层面调度——请求一完成就释放槽位、立刻插入下一个，吞吐比静态批高 10–20 倍

**5. Prefill 与 Decode 分离（2026 的前沿调度）**

- Chunked prefill（分块预填充）：把长 prompt 切块，防止它阻塞 decode
- PD 分离（disaggregation）：prefill 和 decode 跑在不同 GPU 上——一个算力密集、一个显存密集，分开才能各自打满
- 代表：NVIDIA Dynamo、llm-d（K8s 原生分布式推理）；跨引擎传 cache 的 NIXL/LMCache（呼应第三章的网络层）

# 第八章 模型瘦身

> 主题：第七章是"更聪明地用显存"，这一章是"让模型本身变小"。核心判断：优化与否的差距，比 GPU 换代的差距还大。

**1. 量化——把权重和激活值用更少的比特表示**

- 精度阶梯：FP16/BF16 → FP8 → INT4/FP4，位数越低越省显存和带宽，挑战是不掉精度
- 训练后量化（PTQ）主流算法：GPTQ（用二阶 Hessian 信息逐列量化补偿误差）、AWQ（激活感知，保护重要权重）、SmoothQuant（把激活的离群值难度转移到权重）
- 2026 的新前沿：FP4 格式（NVFP4/MXFP4）——Blackwell 原生支持，DeepSeek-R1、Llama-405B 都有 FP4 版本；以及 MR-GPTQ 这类专为 FP4 设计的变体
- 量化感知训练（QAT）：在训练中就模拟量化，用于低精度下恢复精度
- 一个值得讲的判断：难点在"激活值的离群值"——少数极端大的值毁掉整个量化，各算法本质都在和它斗

**2. 蒸馏——把大模型的能力压进小模型**

- 师生范式：用大模型（teacher）的输出训练小模型（student）
- 和推理的关系：蒸馏出的小模型既能独立部署，也能当投机解码的"草稿模型"（接第九章）

**3. 剪枝与结构优化**

- 剪枝：去掉不重要的权重/神经元；NAS（神经架构搜索）自动找更优结构
- 代表：NVIDIA Minitron/Puzzletron——剪枝 + 蒸馏 + FP8 组合拳，实测约 1.64× 吞吐、2.6× 显存压缩

**4. MoE（混合专家）——用稀疏激活换有效容量**

- 原理：每个 token 只路由到一小部分专家（参数），总参数巨大但单次激活的算力小
- 代表模型：Mixtral、DeepSeek、Qwen-MoE
- 推理难点：专家要下放到 CPU 内存时，PCIe 上的 I/O 成为瓶颈——这是当前一批论文（MoE-SpeQ、SP-MoE、SpecMoEOff）在攻的点：用投机执行预取专家、把 I/O 藏到计算背后

# 第九章 投机解码

> 主题：单独成章，因为它是"拿算力换延迟"这一思路最优雅的代表，而且 2026 已是生产标配，方法演进得很清楚。

**1. 基本原理**

- 病根：自回归解码一次只出一个 token，是串行的，GPU 利用率低
- 解法：小而快的草稿模型先猜 N 个 token，大模型一次并行验证这 N 个——猜对就赚、猜错就退回，结果与大模型逐个生成完全等价（无损）
- 本质：用冗余算力（并行验证）换更低延迟

**2. 方法演进（三代，可作为一条清晰的技术史线）**

- 草稿-目标双模型：最经典，需要一个独立的小模型当草稿
- Medusa：不用独立草稿模型，在目标模型上加多个轻量"解码头"各预测一个未来位置，树状注意力验证；实测 2.2–3.6× 加速
- EAGLE 系列：在目标模型的内部特征层（而非 token 层）做自回归预测；EAGLE-1 约 2.7–3.5×，加上动态草稿树（EAGLE-2）和进一步优化（EAGLE-3）达 3–6.5×
- 其他：ReDrafter、Multi-Token Prediction（MTP，DeepSeek 用）、lookahead、n-gram

**3. 工程权衡（本章的"科学"落点）**

- 接受率 vs 草稿成本：草稿模型越强猜得越准（接受率高），但本身越慢——存在最优点
- 与显存带宽的关系：投机解码本质是把 memory-bound 的 decode 变得更 compute-bound，所以它在 GPU 闲置时收益最大（正好解释了为什么它和 MoE 下放场景天然契合）

# 第十章 推理引擎生态

> 主题：前几章是"技术"，这一章是"谁把这些技术打包成了能用的系统"，以及选型背后的工程权衡。

**1. 三大主流引擎及其定位差异**

- vLLM：PagedAttention 的发源地，模型支持广、冷启动快（约 60 秒），上生产最快的路径；适合需要灵活性和快速部署
- TensorRT-LLM：NVIDIA 的数据中心级框架，编译后吞吐最高（高并发下比 vLLM 约快 13%），但冷启动慢（约 28 分钟）；适合单模型、长期运行的生产部署；支持最广的投机解码方法和混合精度量化（FP8/FP4/INT4 AWQ/INT8 SmoothQuant）
- SGLang：擅长共享前缀场景（chatbot、RAG、多轮）的自动前缀缓存
- 一个有用的判断：优化与否的差距大于 GPU 换代的差距——FP8 量化 + Flash Attention 3 + 连续批处理 + 投机解码叠起来，比朴素 FP16 静态批的成本效率高 5–8 倍

**2. 服务化的外围层**

- 前缀缓存（prefix caching）：复用公共 prompt 前缀，多轮/RAG 场景省大量重复计算
- API 网关、路由、流式输出、多 GPU 张量并行
- 部署形态：K8s 原生（llm-d）、存算分离的"AI 工厂"架构

**3. 一条贯穿的趋势**

- 推理基础设施正在走向"开放、标准化、存算解耦"——把 KV cache 的移动抽象成标准 API 层（NIXL），让存储和计算各自独立扩展

# 第十一章 推理的标度关系

> 主题：推理层的收尾，也是最能接你"CyberSpace / 物理学姿态"的一章——把推理当成一个有自身规律的系统去找它的守恒量和权衡曲线。

**1. 三角权衡：成本、延迟、吞吐**

- 三者不可兼得的结构；批越大吞吐越高但单请求延迟越高；延迟敏感（对话）vs 吞吐敏感（批处理）是两套不同的优化目标
- prefill-bound（长上下文 RAG）和 decode-bound（长生成）是两类不同的优化问题，不能一概而论

**2. 推理成本 anchor（让读者拿到 stake）**

- 公开估算：ChatGPT 高峰日成本数百万美元量级
- 单次 query 成本：GPT-4o 一次复杂调用约 $0.01–$0.05；Claude 3.5 / Gemini 同一量级
- API 价格三年降 92%——同等质量的模型，每 token 成本下降一个数量级以上
- 一家中等规模公司部署自己的 LLM 服务：硬件 capex + 电费 + 运维，年成本千万美元起跳
- **判断**：推理优化的每一个百分点，都直接换成现金

**3. 新的标度量：tokens per watt**

- 取代纯 FLOPs 成为效率标尺，因为它同时捕捉"基础设施效率"和"工作负载优化"
- 把第四章的"电力"和本层的"推理优化"统一进同一个指标——这是全篇两层之间的桥
- 行业基准在不断更新：Blackwell + FP4 + continuous batching 的最优实践能跑到 H100 baseline 的 5–10×

**4. 测试时计算（test-time compute）：推理从"跑模型"变成"决定模型多强"**

- "让模型多想一会儿"把推理算力变成一个可以换取能力的维度
- OpenAI o1/o3、DeepSeek R1、Anthropic extended thinking 是这条路的代表
- 工程实现：长 reasoning trace、多次采样后聚合（best-of-N）、tree search、self-refine
- 经济含义：推理时多花 10× 算力 ≈ 模型能力上一个台阶。这把"推理优化"从"省钱"提升到"决定能力"的层级
- 跟训练态的对话：训练时增加算力 vs 推理时增加算力，两条 scaling law 现在并存。哪条 ROI 更高是 2026 业界最大的开放问题之一
- 这是模型/工程边界消融的最清晰落点，也是引向第二篇（后训练）的钩子

**5. 收尾判断**

- 推理优化如何反过来决定"模型实际可用的能力"——同一套权重，优化与否是两个不同的产品
- 留钩子：运行态（推理）讲完，下一篇进入改造态（后训练）与自主态（Agent）
